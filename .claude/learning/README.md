# 🧠 弘爺漢堡 AI Agent 自主學習系統

此目錄用於儲存 AI Agent 的學習資料，實現指令的持續優化與進化。

## 📁 目錄結構

```
.claude/learning/
├── README.md              # 本說明文件
├── feedback/              # 使用者反饋記錄
│   └── [YYYY-MM]/        # 按月份組織
├── improvements/          # 改進記錄
│   └── [command_name]/   # 按指令組織
├── analytics/             # 使用分析數據
│   └── monthly/          # 月度報告
├── templates/             # 學習模板
└── config.json           # 系統配置
```

## 🔧 學習系統指令

| 指令 | 功能 | 使用時機 |
|------|------|----------|
| `/learn` | 記錄使用反饋 | 使用指令後提供回饋 |
| `/improve` | 分析並改進指令 | 發現指令需要優化時 |
| `/newprompt` | 創建新指令 | 需要新功能時 |
| `/promptlog` | 查看使用日誌 | 分析使用情況 |
| `/promptanalyze` | 深度分析指令 | 評估指令效能 |
| `/promptlib` | 管理指令庫 | 查詢/搜尋指令 |

## 📊 學習機制

### 1. 反饋收集
- 每次使用後可透過 `/learn` 提供反饋
- 系統記錄評分、具體意見、改進建議
- 反饋自動歸檔並追蹤處理狀態

### 2. 自動優化觸發
| 觸發條件 | 自動行動 |
|----------|----------|
| 同一指令累積 3+ 負面反饋 | 自動發起優化分析 |
| 同一建議出現 5+ 次 | 納入指令更新計畫 |
| 平均評分低於 3.0 | 標記為緊急優化 |
| 新功能需求 3+ 次 | 建議創建新指令 |

### 3. 持續改進循環

```
收集反饋 → 分析模式 → 識別問題 → 設計改進 → 實施更新 → 驗證效果
     ↑                                                      ↓
     └──────────────────────────────────────────────────────┘
```

## 📈 效能追蹤指標

### 核心指標
- **使用頻率**：各指令的使用次數
- **滿意度**：使用者評分 (1-5)
- **完成率**：成功產出可用內容的比率
- **改進率**：優化後的效能提升

### 健康度指標
- **活躍指令比**：過去 30 天有使用的指令佔比
- **平均滿意度**：所有反饋的平均評分
- **改進完成率**：已處理的改進需求佔比

## 🔄 版本控制

每次指令更新都會記錄：
- 版本號（V1.0, V1.1, V2.0...）
- 更新日期
- 更新內容摘要
- 更新原因（反饋/主動優化/新需求）
- 效果評估（更新後的滿意度變化）

## 📝 使用範例

### 提供反饋
```
/learn aida
```
然後按照提示填寫滿意度和具體意見。

### 改進指令
```
/improve rfm
```
系統會分析現有指令並提出改進建議。

### 創建新指令
```
/newprompt 我需要一個用於設計會員生日活動的指令
```
系統會根據需求設計並創建新指令。

### 查看分析
```
/promptlog all
/promptanalyze aida
```

## ⚙️ 系統配置

編輯 `config.json` 可調整：
- 自動優化觸發閾值
- 反饋記錄保留期限
- 報告生成頻率
- 通知設定
